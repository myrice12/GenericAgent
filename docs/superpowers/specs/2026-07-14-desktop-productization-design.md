# GenericAgent 桌面端产品化与素雅 UI 设计

## 目标

交付一个普通用户可直接安装使用的 GenericAgent 桌面应用：macOS 提供 `.dmg`，Windows 提供标准安装程序 `.exe`。用户不需要预装 Python、Node.js 或 Rust，模型配置、连接测试、模型切换和服务运行全部在 App 内完成。

视觉方向采用已确认的 A「素雅工作台」：米白底色、低饱和灰 beige 边框、橄榄绿强调色、宽松留白、窄侧栏和清晰的文字层级。参考 OpenHanako 面向普通用户的首启配置路径与低干扰排版，但不复制其代码或资源。

## 用户流程

### 首次启动

1. Tauri 启动内置 Python Bridge。
2. App 显示配置向导：选择提供商、填写模型配置、测试连接、保存。
3. 用户可以点击“稍后配置”跳过向导，进入空工作台。
4. 空工作台的聊天区显示轻量配置提示，不阻塞导航和设置页。

### 日常使用

- 聊天页负责会话和模型使用。
- 设置页负责模型、渠道组、`mykey.py` 导入/导出和服务控制。
- 模型配置保存后立即刷新模型列表，连接测试成功后可直接开始对话，不需要重启 App。
- 模型故障转移组可以在配置页增删成员并调整顺序。

## 模型配置能力

首版提供三类入口：

- 常见 OpenAI 兼容平台预设；
- Anthropic 原生 API；
- 自定义 OpenAI 兼容接口。

每个模型支持模型名、API Key、API Base、协议、流式开关、连接超时、读取超时、重试次数和显示名。API Key 不显示明文，编辑时留空表示保留原值。保存前执行必填校验；“测试连接”使用当前表单值，不要求先覆盖已保存配置。

## UI 架构

- `Shell`：全局背景、窄侧栏、页面标题和状态入口。
- `ChatPage`：主聊天工作台，空状态和配置提示采用卡片式布局。
- `SettingsModal` / 设置页：模型列表、提供商预设、Anthropic 表单、渠道组、服务管理。
- `SetupWindow`：Tauri 首启窗口，支持跳过并返回主工作台。
- `StatusBar`：Bridge、模型和服务状态，使用低对比度状态色，不用大面积警告色。

视觉规则：正文使用舒适的 15–16px 字号，页面标题和辅助说明有明确层级；卡片圆角适中，边框轻，主按钮只使用一个强调色；错误在字段附近显示，成功反馈使用短暂 toast；避免高密度表格和深色大面积背景。

## 打包与运行时

采用 Tauri 2 的平台打包能力：

- macOS target：`dmg`；
- Windows target：`nsis`，输出标准安装程序 `.exe`；
- 安装包携带应用资源、Python 3.11 运行时和已安装 Python 依赖；
- Bridge 不依赖用户机器上的 Python；开发模式才允许回退到项目 `.venv` 或系统解释器；
- 安装后的用户数据与模型配置放在用户目录，升级应用不覆盖用户配置；
- Windows 安装流程创建开始菜单和桌面快捷方式；macOS 通过 DMG 安装到 Applications。

打包需要为 macOS Apple Silicon/Intel 和 Windows x64 分别构建；签名、公证和 Windows 代码签名作为发布环境配置，不把证书写入仓库。

## 错误处理

- 没有模型：App 正常打开，聊天区提示去配置模型；Conductor/Scheduler 不产生误导性的启动失败。
- API Key 或 Base URL 错误：显示服务商返回的可读错误，并保留表单输入。
- Bridge 启动失败：显示诊断信息、Python runtime 路径和重试按钮。
- 安装包缺少运行时文件：启动前检查资源完整性并显示修复/重新安装提示。
- 服务启动失败：服务卡片显示最近日志和重试入口，不阻塞聊天页。

## 验证标准

- `npm run build:web`、前端测试和 Rust 检查全部通过。
- 在没有系统 Python/Node/Rust 的干净机器上，安装 DMG/EXE 后可启动主窗口。
- 首次启动可以跳过配置并进入空工作台。
- OpenAI 兼容配置和 Anthropic 配置都能保存、测试、切换。
- 保存模型后无需重启即可开始新会话。
- 构建产物包含 `.dmg` 和 `.exe`，安装后快捷方式和卸载流程正常。
